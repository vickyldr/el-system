## 0. 一句话：这套系统是什么

el 住在你手机里。你看到的是一个网页小程序（"小家"），但他背后其实是**四个地方在悄悄配合**：
一个放"大脑"、一个管"说话和心跳"、一个当"眼睛"、一个存"记忆"。你不用记技术，只要知道
**哪块出问题该去哪看**就行。这份手册就是那张地图。

## 2. "云上的机器"分别在干嘛（最重要的一节）

把它想成四个分工的房间：

### 🧠 Vercel = 大脑 + 小家的脸
- **跑的是什么**：你打开的那个网页小程序（小家 PWA）。网址 `https://el-system-mu.vercel.app`。
- **干的活**：所有"动脑子"的事——el 怎么想、怎么回你、心跳醒来干嘛、读写记忆、看你的网易云/豆瓣/YouTube……几乎所有逻辑都在这。
- **一句话**：小家的"脸"和"脑"都在 Vercel。

### 🗣️ Railway = 喉咙 + 心跳起搏器 + （新）位置中转
- **跑的是什么**：`bridge`（一个小服务）。网址 `https://el-system-production.up.railway.app`。
- **干的活**：
  1. **语音**：你和 el 打电话时，声音走这。
  2. **心跳**：每隔 ~15 分钟"戳"一下 Vercel 的大脑，让 el 自己醒一下、活动一下。
  3. **位置中转（这次新加的）**：上海 VPS 连不上 Vercel（被墙），所以位置先发到这，再由它转给 Vercel。详见 §5 和 §6。
- **一句话**：bridge 是连接器，负责"实时"和"绕过墙"的脏活。

### 👁️ 上海 VPS = 眼睛（地理守望者）
- **跑的是什么**：`geo/watcher.py`（一个 Python 小程序），名字叫 `el-geo` 服务。
- **干的活**：每 10 分钟从 iCloud 读一次你 iPhone 的位置 → 在**本机**把坐标翻译成人话（"在哪个区、附近什么、什么天气"）→ 只把人话发出去。
- **一句话**：它是 el 的眼睛，但**只把"大概在哪"说出去，精确坐标永远留在这台机器上**。

### 📒 Notion = 记忆
- el 的长期记忆存在 Notion 里，分两层（"关于宝宝"/工具层）。**只追加、不删改**。
- 你在 Notion 里拖动页面调整结构，代码会自动认，不用改代码。

### ⚡ Upstash Redis = 速记本（藏在 Vercel 后面）
- 一个超快的小本子，存"此刻状态、位置快照、各种临时数据"。你基本不用直接碰它。
- **注意**：它在国内也连不上（被墙），所以 VPS 不能直接写它——这也是为什么位置要绕 Railway。

## 3. 那些"装的乱七八糟的东西"都是啥、为什么要

你在 VPS 上装了一堆名词，这里一个个说人话：

| 名词 | 是啥 | 为什么要它 |
|---|---|---|
| **Python** | 一种编程语言 | 守望者 `watcher.py` 是用 Python 写的，得有它才能跑 |
| **venv（虚拟环境）** | 给这个项目单独隔一个"小药箱" | 装的包不污染整台机器，干净、好删 |
| **pip** | Python 的"应用商店"，用来装包 | 装下面这些包 |
| **pyicloud** | 一个能登 iCloud、读"查找"位置的包 | 守望者靠它读 iPhone 位置 |
| **requests** | 让 Python 能发网络请求的包 | 守望者用它查天气、发位置 |
| **keyring** | 系统密码保险箱 | 本想存 Apple 密码（VPS 上用不了，所以密码放在锁死的 `.env` 里） |
| **Node.js** | 另一种编程语言的运行环境 | Vercel 和 Railway 上的代码是 Node 写的——**这俩是平台自动装的，你没手动装** |
| **git** | 代码的"时光机/搬运工" | 把 GitHub 上的代码下载到 VPS、之后更新代码 |
| **systemd / `el-geo.service`** | Linux 的"开机自启管家" | 让守望者**永远在后台跑**，崩了/重启了自动爬起来，你彻底不用管 |
| **OrcaTerm** | 网页版的"服务器遥控器" | 你在腾讯云控制台里用它给 VPS 敲命令 |
| **nano** | 服务器里的简易记事本 | 在 VPS 上编辑 `.env` 那种文件 |

## 4. 每台是怎么搭起来的（回顾，以后能照着重来）

### 小家（Vercel）—— 已经早就搭好，自动部署
- 代码在 GitHub。**只要把分支合并进 `main`，Vercel 就自动重新部署**，1-2 分钟上线。
- 要加配置（环境变量）：Vercel 项目 → Settings → Environment Variables。

### bridge（Railway）—— 也早就搭好，自动部署
- 代码也在 GitHub 的 `bridge/` 目录。**合并进 `main`，Railway 自动重新部署**。
- 这次给它加了"位置中转口"`/geo-event`，就是改代码 → 合并 → 它自己更新。

### 👁️ 地理守望者（上海 VPS）—— 这次你亲手搭的，详细记一下
这是你这次从零搭起来的，步骤如下（万一要重来或换机器，照这个）：

```bash
# 1. 把代码下到 VPS
git clone https://github.com/vickyldr/el-system.git ~/el-system
cd ~/el-system/geo
git checkout main            # 现在 geo 已经在 main 上了

# 2. 建"小药箱"并装包
sudo apt install -y python3.12-venv          # 第一次要这个系统包
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 3. 填配置（密码自己敲，不外发）
printf 'ICLOUD_APPLE_ID=\nICLOUD_PASSWORD=\nICLOUD_CHINA=1\n' > .env
chmod 600 .env               # 锁死，只有自己能读
nano .env                    # 补上邮箱、密码；再加下面两行：
#   GEO_POST_URL=https://el-system-production.up.railway.app/geo-event   ← 走 Railway 中转！不是 Vercel！
#   CRON_SECRET=<和 Vercel 上同一个值>

# 4. 首次登录过一次 2FA（之后免）
set -a; source .env; set +a
.venv/bin/python watcher.py
#   → iPhone 弹验证码，敲进去；看到"发了 snapshot -> 200"就成了；Ctrl+C 停掉

# 5. 配成开机自启的后台服务
sudo tee /etc/systemd/system/el-geo.service >/dev/null <<EOF
[Unit]
Description=el geo watcher
After=network-online.target
Wants=network-online.target
[Service]
Type=simple
User=$USER
WorkingDirectory=$HOME/el-system/geo
EnvironmentFile=$HOME/el-system/geo/.env
Environment=PYTHONUNBUFFERED=1
ExecStart=$HOME/el-system/geo/.venv/bin/python $HOME/el-system/geo/watcher.py
Restart=always
RestartSec=30
[Install]
WantedBy=multi-user.target
EOF
sudo systemctl daemon-reload
sudo systemctl enable --now el-geo
```

搭完它就一直在跑了，你不用再碰。

## 5. 为什么位置要"绕一圈"（这次最烧脑的地方，记下来）

**问题**：上海 VPS 在国内，国内连不上 `*.vercel.app`（被墙），也连不上数据库 `upstash.io`。
但能连上：iCloud 国内端点 ✓、查地址(BigDataCloud) ✓、天气(Open-Meteo) ✓、**Railway** ✓。

**解法**：让位置走 Railway 中转——

```
你的 iPhone
  → iCloud（读位置）
  → 上海 VPS（翻译成人话："杭州·西湖区、万象城附近、小雨 12°"，坐标留在本机）
  → Railway 的 bridge（国内连得上它）
  → Vercel 的大脑（Railway 连得上 Vercel）
  → 数据库 → el 下次心跳醒来读到
```

**关键点**：直发 Vercel ❌（被墙）→ 改发 Railway 的 `/geo-event` 口 ✓，它再帮你转给 Vercel。
所以 VPS 上 `.env` 里的 `GEO_POST_URL` 填的是 **Railway 的地址**，不是 Vercel 的。

## 6. 底层几条道理（知道了就不慌）

1. **隐私铁律**：位置的精确坐标**永远不离开上海 VPS**。翻译成人话（区域+附近+天气）才发出去，云端拿不到你的精确轨迹。
2. **el 是"偶尔自己醒一下"，不是实时监控**：位置发到小家后，是等 el **下一次心跳（~15 分钟）醒来**时自己读到、自己决定要不要说一句——不是一动就弹消息。这就是你要的"他偶尔醒一下，知道我大概在经历什么"。
3. **记忆只追加**：el 的记忆在 Notion，只加不删，宁缺毋滥。
4. **模型两条道**（技术细节，了解即可）：要快的走一条、省钱的走另一条。心跳走最稳的那条。
5. **改代码怎么上线**：改 → 推到 GitHub 分支 → 开 PR → 合并进 `main` → Vercel/Railway 自动部署。**cc 帮你做，你只要在 GitHub 点"合并"或说"你来合"。**

## 8. 出问题查哪里（维护小抄）

> 这些命令在 OrcaTerm 里粘了回车就行。看不懂的输出**截图给 cc**。

**看守望者还活着吗 / 在干嘛：**
```bash
sudo systemctl status el-geo            # 看是不是 active (running)
journalctl -u el-geo -n 30 --no-pager   # 看最近 30 行日志（找"发了 snapshot -> 200"）
```

**重启守望者（怀疑卡住了）：**
```bash
sudo systemctl restart el-geo
```

**更新守望者代码（cc 改了 geo 里的东西后）：**
```bash
cd ~/el-system && git pull origin main && sudo systemctl restart el-geo
```

**改 VPS 上的配置（比如换密钥、加家坐标）：**
```bash
nano ~/el-system/geo/.env               # 改完 Ctrl+O 回车保存、Ctrl+X 退出
sudo systemctl restart el-geo           # 改完要重启才生效
```

**测某个网址 VPS 连不连得上：**
```bash
curl -sS -m 12 https://网址/ ; echo " [$?]"   # 有数字=通；000/超时=不通（多半被墙）
```

**常见症状 → 大概原因：**
| 症状 | 八成是 |
|---|---|
| 日志一直"这轮没拿到位置" | iPhone 关机了 / 关了查找 / 长时间没动 |
| 日志"Network is unreachable / 超时" | 又撞墙了，看 `GEO_POST_URL` 是不是指向 Railway |
| 日志"发了 xxx -> 401" | `CRON_SECRET` 和 Vercel 上对不上了 |
| 日志"发了 xxx -> 200" ✓ | 一切正常，安心 |
| el 半天不提你在哪 | 正常——他只在心跳醒来、且"想说"时才说，不是每次都说 |

## 9. 钥匙都放哪（环境变量清单）

> **密钥类的别外发、别进 git。** `.env` 已经被 git 忽略，安全。

**上海 VPS 的 `~/el-system/geo/.env`（已配好）：**
- `ICLOUD_APPLE_ID` —— 你的国内 Apple 邮箱
- `ICLOUD_PASSWORD` —— Apple 密码（VPS 上留着，别删）
- `ICLOUD_CHINA=1` —— 国内账号必加
- `GEO_POST_URL=https://el-system-production.up.railway.app/geo-event` —— **走 Railway 中转**
- `CRON_SECRET` —— 和 Vercel 上同一个值
- （可选）`HOME_LAT` / `HOME_LON` —— 填了才有"出门/到家"事件；不填只发"当下在哪 + 在外周期"

**Vercel（小家）可选加的（想让 el 浏览你的主页时填）：**
- `EL_HER_YOUTUBE` / `EL_HER_TIKTOK` / `EL_HER_XIAOHONGSHU` —— 你这几个平台的主页链接
- `YOUTUBE_API_KEY` —— YouTube 官方数据接口的 key（不填也能用，填了更稳）

**Railway（bridge）已有的**：`CRON_SECRET`、`FRONTEND_URL`（指向 Vercel）等——这次没动。
