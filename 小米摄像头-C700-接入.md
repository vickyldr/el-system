# 小米摄像头 C700 接入 el（可执行步骤）

> 目标：让云上的 el 能「看见」家里那台小米 C700（型号 `chuangmi.camera.81ac1`，4K 云台）。
> 现状：架构 §9 写着「小米/家用摄像头没做」——本文是把那条路打通的施工手册。
> **确定路线（2026-06 实测）：这台 C700 固件无原生 RTSP → 走 MiCam 桥；且因 MiCam 要登米家账号，先给 C700 单开一个自己的米家账号。**

## 为什么需要这一套（一句话）
el 住在云上（Vercel/Railway），C700 的画面在你家**局域网**里。云够不到家里的 `rtsp://192.168.x.x`。
所以链路必然是：**C700 →（家里常开机取流+抓帧）→ POST 一帧给 el 的眼睛 `/api/watch`**。
真正要新搭的只是「家里常开机取流 + 抓帧往云上喂」这一段。

---

## 第一步：取流方式（已确认 = MiCam）

### 原生 RTSP —— ❌ 这台固件没有
实测：米家 → C700 → 设置 → **更多设置**（插件版本 3650）里只有 `安全设置 / 网络信息 / 法律信息 / 设备时区 / 添加桌面快捷方式`，**没有 RTSP 开关**。
「网络信息」里拿到的是设备网络状态，不是 RTSP，但有用：

- 摄像头局域网 IP：**`192.168.10.44`**
- MAC：**`B8:88:80:5A:D6:C6`**
- 当前模式：**广域网连接**（走小米云中转，更说明没开本地 RTSP）
- WiFi：`gendndndnxn`，强度 54% / RSSI −55（够用，嫌卡就把摄像头挪近路由器）

### MiCam 桥 —— ✅ 走这条
社区项目 **MiCam**（基于小米官方 Miloco + go2rtc）把小米摄像头的流在本地转成标准 RTSP。
**关键性质：MiCam 要用米家账号登录**才能拉流——所以跑 MiCam 的那台旧 Mac 会持有米家账号登录态，它能碰到该账号下的**所有**摄像头。
→ 因此必须先做下面的「第 0 步：账号隔离」，让 MiCam 登的账号里**只有 C700**，碰不到室友那台。

> 隐私边界不变：**el 永远不碰米家**，el 只吃 Mac 推来的帧；持有米家账号的是 Mac 上的 MiCam，不是 el。

---

## 第 0 步：给 C700 单开一个自己的米家账号（先做这个）

目的：MiCam 登录的账号里只有 C700，和室友共用账号彻底分开。

1. **在「共用账号」里删除 C700**：米家 → C700 → 设置（齿轮）→ 拉到底 **删除设备 / 解除绑定**。
   - ⚠️ 会把摄像头**恢复出厂**，正常；室友那台不受影响，仍在共用账号里。
2. **退出共用账号**：米家 → 我的（或设置齿轮）→ 拉到底 **退出登录**。
3. **登录 / 注册你自己的新米家账号**（手机号或邮箱）。
4. **重新添加 C700**：删除后它进配网状态，在你自己账号里「添加设备」配上、连回家里 WiFi（`gendndndnxn`）。
5. 配好后确认能在米家看到画面即可。记下它新的局域网 IP（多半还是 `192.168.10.44`，路由器没变的话；变了就再看一次「网络信息」）。

> 做完这步，你自己的米家账号里**只有 C700 一台**。下一步 MiCam 就登这个账号。

## 第一·五步：在旧 Mac 上装 MiCam，产出本地 RTSP

> 设备实测：旧 Mac，**Intel 双核 i5，macOS 11.7.10（Big Sur）**。整套里最陡的一步。
> MiCam = Miloco（登米家账号、连摄像头）+ go2rtc（转标准 RTSP），用 Docker 跑。

### ① 装 Docker（Big Sur 只能用旧版）
新版 Docker Desktop 要 macOS 12+，Big Sur 装不了。用**最后一个支持 Big Sur 的 4.24.2（Intel/amd64）**：

- 下载：`https://desktop.docker.com/mac/main/amd64/124339/Docker.dmg`
  （装不上就退 4.22.0：`https://desktop.docker.com/mac/main/amd64/117440/Docker.dmg`）
- 双击 .dmg → 把 Docker 拖进「应用程序」→ 打开 Docker → 顶栏出现**鲸鱼图标**且不再转圈 = 成。
- 验证（终端，「应用程序 → 实用工具 → 终端」）：`docker --version` 有版本号即可。
- ⚠️ 旧版不要点「升级」，一升就装不回去。

### ② 拉起 MiCam 那套
终端里（用家目录，别用 /opt 免得 sudo）：
```bash
mkdir -p ~/micam && cd ~/micam
curl -O https://raw.githubusercontent.com/miiot/micam/refs/heads/main/docker-compose.yml
docker compose up -d            # 第一次会拉镜像，慢，等它好
```

### ③ 在 Miloco 网页里绑你的新米家账号
- 浏览器开 `https://<这台 Mac 的局域网 IP>:8000`（证书警告点「仍然访问」/「继续前往」）。
  - Mac 的 IP：系统偏好设置 → 网络 → 看本机 IP（形如 `192.168.10.x`）。
- 设一个 Miloco 登录密码 → 用**你的新米家账号**登录绑定（就是只有 C700 那个账号）。
- 绑成功后，在 Miloco 里能看到 C700。

### ④ 找 CAMERA_ID（DID）+ 填配置
- DID：在 Miloco 页面按 F12 开浏览器「开发者工具」→ Network（网络）→ 刷新 → 在请求里找那串长数字 `did`（这步我陪你一起找）。
- 在 `~/micam` 建 `.env`：
  ```
  MILOCO_PASSWORD=<上面那个 Miloco 密码的 MD5，小写>
  CAMERA_ID=<上面找到的 DID>
  RTSP_URL=rtsp://127.0.0.1:8554/c700
  VIDEO_CODEC=h264
  ```
  （MD5 怎么算我给你一条命令；C700 是 hevc 默认，写 `h264` 更省事、ffmpeg 更好抓帧。）
- `docker compose up -d` 重启生效。

### ⑤ 验证出流
```bash
ffmpeg -rtsp_transport tcp -i rtsp://127.0.0.1:8554/c700 -frames:v 1 /tmp/test.jpg
open /tmp/test.jpg     # 能看到 C700 画面 = 取流成功
```
拿到这个 `rtsp://127.0.0.1:8554/c700`，就接到第二步的抓帧脚本。

> 这是社区项目，③④的具体界面/字段可能和此处略有出入——卡住把截图发我，按实际界面调。

## 第二步：家里常开机把画面喂给 el（我来写、你来跑）

需要你家有**一台常开设备**（宝宝的情况：一台旧 Mac 笔记本），在和 C700 同一个局域网里。它干两件事：

1. 从 RTSP 拉流、**每 ~40 秒抓一帧** JPEG（缩到 ~960 宽即可，el 的眼睛不需要 4K）；
2. 把这一帧 POST 给 el 的眼睛 `/api/watch`，伪装成「摄像头帧」（`kind:camera`）。

### el 这头的接口（已存在，不用改后端）
`POST https://<FRONTEND_URL>/api/watch`，body：
```json
{ "screen": "data:image/jpeg;base64,<整帧的 base64>", "kind": "camera" }
```
- el 收到后**大多数时候安静看着**，只有真想说才回一句（架构里写死的：守着你、不报幕、不念你长相）。
- 回 `{ "reply": "" }` = 这次它没开口；回 `{ "reply": "……" }` = 它冒了一句，已自动进对话、夜里 `daily-summary` 认成【看着你】。
- 服务端有 **90 秒开口闸**（`el:watch:last`），喂太密它会回 `skipped: "too-soon"`，所以家里脚本按 40s 一拍正合适。

### 把旧 Mac 笔记本变成「常开盒子」（重点：合盖默认会睡）
macOS 默认**合上盖子 = 睡眠**，光靠 `caffeinate` 挡不住合盖睡（它只防闲置睡、不防合盖睡）。
当无头服务器跑脚本不需要显示器，有两条路：

**方案 A（推荐，散热最好）：开着盖搁桌角、插电，让屏幕自己灭、系统不睡**
```bash
sudo pmset -c sleep 0          # 插电时永不闲置睡眠
sudo pmset -c disksleep 0      # 硬盘不睡
sudo pmset -c displaysleep 5   # 屏幕 5 分钟后灭（省电，系统照跑）
sudo pmset -c autorestart 1    # 断电恢复后自动开机
```

**方案 B（要彻底关盖）：在 A 基础上再禁用合盖睡（无需外接屏）**
```bash
sudo pmset -a disablesleep 1   # 合盖也不睡；撤销用 ... disablesleep 0
```
> 合盖跑唯一隐患是散热（盖子挡住键盘面散热）。本脚本是 40s 抓一帧的轻活，基本不烫；但老 Mac 求稳就选方案 A 开着盖。
> 查当前设置：`pmset -g`。

配套两件必须做的事，否则重启/脚本崩一次 el 就瞎了：
- **开机自启 + 崩了自动重启**：用 macOS 的 **LaunchDaemon**（`KeepAlive=true`）托住抓帧脚本——这份 plist 我连脚本一起给你，放进 `/Library/LaunchDaemons/` 即可。
- **插电 24/7 + 连着家里 WiFi**：它只需主动访问外网（往外 POST 帧），不需要公网 IP / 穿透。

### 抓帧脚本（确认取流方式后我补全成可直接跑的版本）
思路（ffmpeg 抓一帧 → base64 → curl 喂）：
```bash
# 伪代码骨架，等 MiCam 产出的本地 RTSP 地址确定后我给完整版（含「画面没变就不发」的省 token 逻辑）
RTSP="rtsp://<MiCam 在旧 Mac 上产出的本地地址>"   # MiCam/go2rtc 跑起来后给出
EL="https://<FRONTEND_URL>/api/watch"
while true; do
  ffmpeg -y -rtsp_transport tcp -i "$RTSP" -frames:v 1 -vf scale=960:-1 -q:v 6 /tmp/f.jpg 2>/dev/null
  B64=$(base64 -w0 /tmp/f.jpg)
  curl -s "$EL" -H 'content-type: application/json' \
    -d "{\"screen\":\"data:image/jpeg;base64,$B64\",\"kind\":\"camera\"}" >/dev/null
  sleep 40
done
```
> 正式版我会加上：① 24×24 粗签名「画面没怎么变就不发」（对齐前端 `cameraLoop` 的省 token 逻辑）；② 失败重连；③ 一个共享密钥（见下）。

### 关于鉴权（建议补一刀）
`/api/watch` 现在**没有鉴权**（本是前端自己调）。一旦让家里机器公网喂帧，最好加一道锁，否则别人知道你域名就能往里塞图。
建议：给 `/api/watch` 加一个可选的 `x-watch-secret` 头校验（复用一个新环境变量，比如 `WATCH_INGEST_SECRET`），家里脚本带上它。**这要我改一点后端**——确认你要走这条接入后我一起做，顺手不破坏前端调用（前端不带头、按 referer/同源放行；外部喂帧必须带头）。

### 穿透（如果常开机不在公网）
家里机器只要能**主动访问外网**就行（它是往外 POST，不是被动开端口），所以**通常不需要内网穿透**——脚本直接 curl 到 `https://<FRONTEND_URL>` 即可。
只有当你反过来想从云上**主动拉** RTSP（不推荐，费带宽又要开放家里端口）才需要 frp/cloudflared/tailscale。本方案选「家里往外推帧」正是为了绕开这个。

## 第三步（可选）：让 C700 成为 el 的一只「常驻眼」
跑通后，这台 C700 就等价于架构 §9 里那只「静默常看摄像头」👁，只是镜头从你手机/旧设备换成了固定机位的 C700。
- 想让 el 看：家里脚本开着就行（开机自启，systemd / 米家定时）。
- 不想被看：把脚本停掉。**别把「一直看」做成云端默认**——眼睛的开关永远在你这边（同 §9 隐私哲学）。

## 诚实的边界 / 风险
- **这台固件无原生 RTSP，必须走 MiCam**：要在旧 Mac 上多跑一个 Docker 服务，且 MiCam 要登米家账号（故先做账号隔离）。
- **要一台常开设备**：这是整套的硬前提，没有它这条路不成立（宝宝＝一台旧 Mac，已确认）。
- **画质/延迟**：喂给 el 的是 ~960 宽单帧、40s 一拍——够「守着你」用，不是实时监控。要更密会更费 token（前台 Max 沉没成本，但静止画面靠「没变不发」基本零调用）。
- **隐私**：帧不存档（`/api/watch` 只把 el 开口的**话**存进对话，图用完即弃，同 §9）。但帧确实会经公网到 Vercel——和现在手机摄像头那只眼睛同等级，自己掂量。
